# Chemistry Recipe Optimizer

A Python-based tool for optimizing chemistry recipes in BYOND-based space station simulation games. This tool helps chemists find the most efficient recipes for producing target reagents while minimizing waste, avoiding harmful byproducts, and respecting the game's 5u dispensing mechanics.

## Feature Overview

### Basic Reagents First

The optimizer prioritizes recipes that use **basic reagents** as inputs. Basic reagents are fundamental building blocks that cannot be easily crafted from other chemicals (e.g., Water, Carbon, Oxygen, Hydrogen, Sugar, Ethanol). By preferring basic reagents, the optimizer:

- Reduces dependency chains (fewer intermediate crafting steps)
- Minimizes cumulative resource waste across multiple reactions
- Simplifies the crafting process for players

Each reagent has a **basicness score** (1 = most basic, higher = more complex). The optimizer uses this score to prefer simpler recipes.

### Efficiency Scoring

The optimizer uses a weighted scoring algorithm to evaluate recipes based on multiple factors:

| Factor | Weight | Description |
|--------|--------|-------------|
| Efficiency | 10.0 | Output-to-input ratio (higher is better) |
| Basicness | 2.0 | Preference for basic reagents (inverted score) |
| Safety | -3.0 | Penalty per harmful reagent in inputs |
| Temperature | +/- 5.0 | Bonus for cold recipes if preferred |
| Byproducts | -5.0 | Penalty per byproduct in results |

The **efficiency score** is calculated as:
```
efficiency_score = total_output / total_input
```

Higher scores indicate more efficient recipes that produce more product with less waste.

### Byproduct Minimization

Byproducts are unwanted chemicals produced alongside your target reagent. The optimizer:

- Tracks all byproducts generated by each recipe
- Allows setting a maximum byproduct limit (default: 3)
- Penalizes recipes with more byproducts in scoring
- Marks harmful byproducts with a hazard symbol (☣️) in output

### Reagent Avoidance

You can specify reagents to avoid during optimization. This is useful for:

- Avoiding expensive or rare ingredients
- Skipping recipes that require dangerous chemicals
- Working around supply limitations
- Excluding allergens or unwanted substances

Avoidance works by both reagent name and typepath (e.g., `--avoid Ethanol` or `--avoid /datum/reagent/ethanol`).

## Prerequisites

- **BYOND 515+** (for exporting chemistry data from the game)
- **Python 3.10+** (for running the optimizer)
- **No external dependencies** (uses only Python standard library)

## Setup Instructions

### Step 1: Export Chemistry Data from BYOND

Before using the optimizer, you must export the game's chemistry data to a JSON file.

1. Add the export proc to your codebase (already included in `export_chemistry_data.dm`)
2. Call the proc in-game or during initialization:
   ```dm
   /proc/export_chemistry_data()
   ```
3. The proc exports data to `scripts/chemistry_optimizer/data/chemistry_data.json`

The exported data includes:
- All reagents with properties (name, type, harmful flag, basicness score)
- All chemical reactions with inputs, outputs, and temperature requirements

### Step 2: Run the Optimizer

Navigate to the script directory and run:

```bash
cd scripts/chemistry_optimizer
python recipe_optimizer.py [options]
```

Or run from anywhere with the full path:

```bash
python scripts/chemistry_optimizer/recipe_optimizer.py -r Ethanol -a 50
```

## Usage Examples

### Basic Usage

Produce 50 units of Ethanol:
```bash
python recipe_optimizer.py -r Ethanol -a 50
```

Produce 30 units each of multiple reagents:
```bash
python recipe_optimizer.py -r Ethanol -r "Space Drug" -a 30
```

### Advanced Usage

Avoid specific reagents:
```bash
python recipe_optimizer.py -r Ethanol --avoid Sugar --avoid Carbon
```

Limit byproducts to 1:
```bash
python recipe_optimizer.py -r "Advanced Chemical" --max-byproducts 1
```

Prefer cold recipes:
```bash
python recipe_optimizer.py -r Ethanol --prefer-cold
```

Allow dangerous reactions (disable safe preference):
```bash
python recipe_optimizer.py -r Ethanol --no-prefer-safe
```

Don't prefer basic reagents:
```bash
python recipe_optimizer.py -r Ethanol --no-prefer-basic
```

### JSON Input/Output

Create a request file `request.json`:
```json
{
  "targets": [
    {"reagent": "Ethanol", "amount": 50},
    {"reagent": "Space Drug", "amount": 30}
  ],
  "avoid": ["Sugar"],
  "max_byproducts": 2,
  "prefer_cold": true,
  "prefer_safe": true,
  "prefer_basic": true
}
```

Run with JSON input and output:
```bash
python recipe_optimizer.py -i request.json -o results.json
```

## Basicness System Explanation

The **basicness score** determines how fundamental a reagent is in the chemistry hierarchy:

| Score | Meaning | Examples |
|-------|---------|----------|
| 1 | Most basic (elemental or cannot be crafted) | Water, Carbon, Oxygen, Hydrogen, Sugar, Ethanol |
| 2+ | Crafted reagents (2 + number of recipes that produce it) | Space Drug (5), Advanced Chemical (10) |

### How Basicness is Calculated

```dm
/proc/calculate_reagent_basicness(reagent_type)
    // Check if reagent is in predefined basic list
    if (reagent_type in basic_reagents)
        return 1

    // Count recipes that produce this reagent
    var/produced_count = 0
    for (reaction in GLOB.chemical_reactions_list)
        if (reagent_type in reaction.results)
            produced_count++

    // No recipes = most basic
    if (produced_count == 0)
        return 1

    // More recipes = more complex
    return 2 + produced_count
```

**Why it matters:** Recipes using basic reagents (score = 1) are preferred because they:
- Require fewer intermediate crafting steps
- Have more predictable resource requirements
- Generate less cumulative waste

## 5u Dispensing Rule Explanation

The game's chemistry system uses a **minimum dispensing increment of 5 units (5u)**. All reagent amounts must be multiples of 5.

### How Rounding Works

The optimizer rounds all calculated amounts **up** to the nearest 5u increment:

| Calculated Amount | Rounded Amount |
|-------------------|----------------|
| 3u | 5u |
| 7u | 10u |
| 12u | 15u |
| 23u | 25u |
| 50u | 50u |

### Implementation

```python
MIN_DISPENSE = 5

def round_to_dispense(self, amount: float) -> int:
    """Round amount up to nearest 5u increment."""
    if amount <= 0:
        return 0
    return int(math.ceil(amount / MIN_DISPENSE) * MIN_DISPENSE)
```

### Why This Matters

1. **Input calculations**: When scaling recipes, inputs are rounded up to ensure you have enough reagents
2. **Resource planning**: Always plan for slightly more input than the theoretical minimum
3. **Waste estimation**: The difference between calculated and rounded amounts represents potential waste

### Example

To produce 12u of Ethanol (recipe requires 5u Sugar + 5u Water per batch):
- Batches needed: 12 / 1 = 12 batches
- Sugar needed: 5 × 12 = 60u → rounds to 60u (already multiple of 5)
- Water needed: 5 × 12 = 60u → rounds to 60u

## Algorithm Description

The optimizer uses a **greedy scoring algorithm** to select the best recipe for each target reagent:

### Step 1: Find Candidate Recipes

For each target reagent, find all recipes that produce it:
```python
recipes = self.find_recipes_for_reagent_name(reagent_name)
```

### Step 2: Filter by Constraints

Remove recipes that:
- Use avoided reagents (by name or typepath)
- Exceed the maximum byproduct limit

### Step 3: Score Each Recipe

Calculate a weighted score for each candidate:
```python
score = (efficiency × 10.0)
      + ((10.0 - basicness) × 2.0)  # Inverted so lower basicness = higher score
      - (harmful_count × 3.0)
      + (cold_bonus if prefer_cold else 0)
      - (byproduct_count × 5.0)
```

### Step 4: Select Best Recipe

Sort by score (descending) and select the highest-scoring recipe:
```python
scored_recipes.sort(key=lambda x: x[0], reverse=True)
best_recipe = scored_recipes[0][1]
```

### Step 5: Calculate Inputs

Scale the recipe to produce the target amount, rounding to 5u increments:
```python
batches_needed = target_amount / len(results)
for reagent_type, base_amount in required_reagents.items():
    scaled_amount = base_amount * batches_needed
    rounded_amount = round_to_dispense(scaled_amount)
```

### Step 6: Aggregate Results

Combine results from all target reagents:
- Sum total input and output amounts
- Collect all byproducts
- Aggregate input reagents (combining duplicates)
- Calculate overall efficiency score

## Troubleshooting

### "No recipes found for: [Reagent Name]"

**Cause:** The reagent cannot be crafted (it's a basic reagent like Water or Oxygen).

**Solution:** This reagent is a basic building block. You need to obtain it from other sources (vending machines, mining, etc.) rather than crafting it.

### "No valid recipes after filtering"

**Cause:** All recipes for the target reagent were excluded due to:
- Avoided reagents in the recipe
- Too many byproducts
- Other constraints

**Solution:** 
- Remove some items from the avoid list
- Increase `--max-byproducts`
- Check if the reagent has any valid recipes at all

### "Optimization FAILED"

**Cause:** No recipes could be found for one or more target reagents.

**Solution:** Check the warnings section for specific failure reasons. Common fixes:
- Verify reagent names are spelled correctly (case-insensitive)
- Ensure chemistry data was exported correctly
- Check that the reagent is craftable

### Amounts seem too high

**Cause:** The 5u rounding rule may cause amounts to be higher than expected.

**Solution:** This is intentional. The optimizer rounds up to ensure you have enough reagents. Plan for the rounded amounts.

### JSON parsing errors

**Cause:** Malformed JSON in input file.

**Solution:** Validate your JSON file using a JSON validator. Common issues:
- Missing commas between array elements
- Unquoted string values
- Trailing commas

## Limitations and Future Enhancements

### Current Limitations

1. **Single-path optimization**: Each target is optimized independently; no cross-target optimization for shared intermediates
2. **No catalyst handling**: Catalysts are tracked but not optimized
3. **Temperature ranges**: Only cold/hot preference, no optimal temperature optimization
4. **No inventory awareness**: Doesn't consider existing reagent stockpiles
5. **Static data**: Requires manual re-export when game chemistry changes

### Planned Enhancements

1. **Multi-target optimization**: Find optimal paths when producing multiple reagents that share intermediates
2. **Catalyst optimization**: Include catalyst availability and reuse in scoring
3. **Temperature optimization**: Prefer recipes within achievable temperature ranges
4. **Inventory integration**: Accept current inventory as input to minimize new resource requirements
5. **Incremental updates**: Support partial data updates when only some recipes change
6. **Recipe chaining**: Automatically expand intermediate reagents to show full dependency chains
7. **Cost optimization**: Factor in reagent costs (if available) for economic optimization
8. **Batch optimization**: Suggest optimal batch sizes to minimize 5u rounding waste
